# System Requirements

BizNode is designed to run on minimal hardware, but certain requirements must be met for optimal performance.

---

## Hardware Requirements

### Minimum Requirements

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| **USB Storage** | 32GB | 64GB+ |
| **RAM** | 8GB | 16GB |
| **CPU** | 4 cores | 8+ cores |
| **Internet** | Required for Telegram/Email | Stable connection |

### Recommended Setup

```
┌─────────────────────────────────────────────────────────────┐
│                    RECOMMENDED SETUP                        │
├─────────────────────────────────────────────────────────────┤
│  Hardware:                                                  │
│  • 64GB USB 3.0/3.1 flash drive                            │
│  • Host machine: 16GB RAM, 8-core CPU                     │
│  • Reliable internet connection                            │
├─────────────────────────────────────────────────────────────┤
│  Software:                                                  │
```

## Software Requirements

### Docker

BizNode runs entirely in Docker containers.

**Installation:**

- **Windows**: [Docker Desktop](https://www.docker.com/products/docker-desktop)
- **Mac**: [Docker Desktop](https://www.docker.com/products/docker-desktop)
- **Linux**: 
  ```bash
  curl -fsSL https://get.docker.com | sh
  sudo systemctl enable docker
  sudo systemctl start docker
  ```

**Minimum Docker Version:** 20.10+

```bash
docker --version
# Should output: Docker version 20.10.0 or higher
```

### Docker Compose

Included with Docker Desktop. For Linux:

```bash
docker-compose --version
# Should output: docker-compose version 1.29.0 or higher
```

## Ollama Requirements

### What is Ollama?

Ollama is the local LLM runtime that powers BizNode's reasoning capabilities.

### Model Requirements

| Model | Size | Purpose | RAM Usage |
|-------|------|---------|-----------|
| **qwen2.5** | 4GB | Main reasoning | ~6GB |
| **nomic-embed-text** | ~500MB | Embeddings | ~1GB |

### Testing Ollama

```bash
# After docker compose up
docker compose exec ollama ollama list

# Should show:
# NAME                ID          SIZE      MODIFIED
# qwen2.5:latest      abc123...   4.5GB     2 days ago
```

### Memory Optimization

If running on limited RAM:

```bash
# Pull smaller model
docker compose exec ollama ollama pull llama3:8b

# Adjust .env
LLM_MODEL=llama3:8b
```

## Qdrant Requirements

### What is Qdrant?

Qdrant is a vector database for semantic search and similarity matching.

### Storage Needs

| Data Type | Approximate Size |
|-----------|------------------|
| 1,000 business notes | ~50MB |
| 10,000 business notes | ~500MB |
| 100,000 business notes | ~5GB |

### Persistent Storage

Qdrant data is stored in `./qdrant_storage/` on the host:

```bash
# Check storage usage
du -sh qdrant_storage/

# Backup
cp -r qdrant_storage/ backup_qdrant_$(date +%Y%m%d)/
```

## Network Requirements

### Required Internet Access

BizNode requires internet for:

| Service | Purpose | Bandwidth |
|---------|---------|-----------|
| **Telegram API** | Bot communication | Low |
| **SMTP** | Email notifications | Low |
| **Ollama Download** | Initial model download | High (one-time) |

### Offline Operation

Once running, BizNode can operate with limited connectivity:

```
Online Required For:
├── Telegram message send/receive
├── Email delivery
└── Model updates (optional)

Offline Capable:
├── LLM reasoning (local)
├── Vector search (local)
├── Database queries (local)
└── Lead processing (local)
```
